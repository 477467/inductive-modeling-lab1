# Лабораторна робота: Лінійна регресія

Цей проєкт генерує синтетичні дані, навчає модель лінійної регресії (`scikit-learn`), обчислює метрики (MSE, MAE, R^2) та будує графік.

## Вимоги
- Windows 10/11
- Встановлений Python (рекомендовано через лаунчер `py`, версія 3.12+ або 3.13)

## Структура
- `lab1.py` — основний скрипт
- `requirements.txt` — залежності
- `run.ps1` — скрипт PowerShell для запуску
- `run.bat` — бат-файл для запуску подвійним кліком

## Швидкий старт (подвійний клік)
1. Двічі клікніть `run.bat` у папці проєкту.
   - Скрипт сам:
     - створить та активує `venv` (`.venv/`), якщо його ще немає,
     - встановить/оновить залежності,
     - увімкне UTF-8 для коректного виводу в консолі,
     - запустить `lab1.py`.

Якщо подвійний клік заблоковано політикою виконання, натисніть правою кнопкою `run.bat` → "Run as administrator" або скористайтесь командою нижче у терміналі.

## Запуск через PowerShell (рекомендовано)
Відкрийте PowerShell у папці проєкту та виконайте:

```powershell
powershell -ExecutionPolicy Bypass -File .\run.ps1
```

## Ручний запуск (альтернатива)
```powershell
# 1) Створити віртуальне середовище (разово)
py -m venv .venv

# 2) Активувати середовище
.\.venv\Scripts\Activate

# 3) Встановити залежності
pip install -r requirements.txt

# 4) Увімкнути UTF-8 і запустити
$env:PYTHONIOENCODING='utf-8'; chcp 65001 > $null; python .\lab1.py
```

## Примітки
- Якщо у консолі з'являється помилка кодування (символи типу `²`), у нас вже замінено на `R^2`. Також `run.ps1` вмикає UTF-8.
- Якщо VS Code просить "Relaunch Terminal" — погодьтеся, це встановлює інтеграцію терміналу і не є помилкою вашого коду.
